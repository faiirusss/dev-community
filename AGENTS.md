## CONVENTIONS

### Imports
- **ALWAYS use `~/` alias**: `import { db } from "~/db"`  
- Never use relative `../../../` imports for src files

### Naming
- **Files:** Components/ui = kebab-case (`avatar.tsx`), Hooks = camelCase (`use-toast.ts`)
- **Database:** snake_case columns (`user_id`, `created_at`), camelCase code (`userId`)
- **Exports:** Named exports preferred (avoid `export default`)

### tRPC
- `publicProcedure` ‚Äî open access
- `protectedProcedure` ‚Äî requires session (throws UNAUTHORIZED)
- Add routers in `src/server/root.ts`

### Database
- Define tables with `pgTable('table_name', { ... })`
- Add relations with `relations(table, ({ one, many }) => ({ ... }))`
- Infer types: `export type User = typeof users.$inferSelect`
- Always export from `src/db/schema/index.ts`

### Routes (TanStack Router)
- `_authenticated.tsx` ‚Äî Route guard, redirects to `/login`
- `_authenticated/dashboard.tsx` ‚Äî Protected route
- `$username.tsx` ‚Äî Dynamic parameter route
- `routeTree.gen.ts` ‚Äî Auto-generated, DO NOT EDIT

## ANTI-PATTERNS

- DON'T modify `routeTree.gen.ts` ‚Äî it's auto-generated by TanStack Router
- DON'T use relative imports (`../../`) when `~/` alias is available
- DON'T use `export default` ‚Äî prefer named exports
- NEVER commit `.env` or `.env.local`
- DON'T run `db:push` in production ‚Äî use migrations only
